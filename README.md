# zeon-assets: Apep Z-Rotations installer + assets

The Windows installer (`apep/install.ps1`) downloads everything from this repo.

## Layout
    apep/
      install.ps1        # the installer (users run it via irm | iex)
      manifest.txt       # auto-generated by the Action below (path <space> sha1-10)
      mmaps-335.json     # small descriptor for the optional pinned MMAPS release
      Z-Rotations.enc    # the built plugin (you commit this)
      icons/*.png icons/*.tga  esp/*.png  fonts/*.ttf fonts/OFL.txt

## Publishing an update
1. Build the plugin: `python tools/bundle_zrotations.py` → `dist/Z-Rotations.lua`,
   then encrypt (Ctrl+Alt+E) → `Z-Rotations.enc`.
2. In the Apep repo, run `python tools/generate_zrotations_asset_manifest.py` to
   confirm the asset upload checklist (every `apep/<path>` must exist here).
3. Copy the assets + the new `Z-Rotations.enc` into this repo under `apep/`.
4. Push. The `generate-apep-manifest` Action (see `generate-apep-manifest.yml`,
   install it as `.github/workflows/…`) regenerates `apep/manifest.txt` with a
   line for every file including `Z-Rotations.enc`.

Users then run:
    irm https://raw.githubusercontent.com/itszeondk/zeon-assets/main/apep/install.ps1 | iex

The installer re-downloads only files whose hash changed.

## Publishing MMAPS 335

MMAPS use a separate release channel because the payload is much larger than
the normal per-file asset manifest. The installer downloads one ZIP from an
immutable GitHub release URL; the small descriptor stays at
`apep/mmaps-335.json` on the assets repo's default branch.

### Reproduce and package the current map set

The current maps were generated with this exact command:

```powershell
mmaps_generator.exe --maxAngleNotSteep 50 --maxAngle 60 --skipLiquid false --skipContinents false --skipJunkMaps true --skipBattlegrounds false --bigBaseUnit false
```

Recorded generation inputs:

- `mmaps_generator.exe` SHA-256:
  `DA68C5D04D28F64BB568B6D93D0C3227551A81687B9388A335D93A498EC1D97D`
- Ascension `Data\patch-B.MPQ` SHA-256:
  `C48159C6D6DDE97ACA850A64B1EF423162F8969E0BB82F01E39BCE7E17C24B2E`
- Payload: 99 `.mmap` + 3,527 `.mmtile` files, 3,540,244,220 bytes
- Format: Detour version 7, MMAP version 15, 20-byte tile headers, and liquids
  enabled (`useLiquids=1`). The header's declared payload length must equal the
  file length minus 20. A header-only map such as map 616 is valid; the
  packager intentionally does not require every `.mmap` to have a tile.

Validate the source and both provenance files without producing the large ZIP:

```powershell
python tools/package_mmaps_release.py --validate-only --source "C:\Users\User\Documents\335" --generator-exe "C:\Users\User\Documents\nav\mmaps_generator.exe" --source-archive "C:\Ascension\Launcher\resources\ascension-live\Data\patch-B.MPQ"
```

Build the deterministic archive and descriptor (the projected archive is about
1.17 GiB, so allow time and sufficient free disk space):

```powershell
python tools/package_mmaps_release.py --source "C:\Users\User\Documents\335" --generator-exe "C:\Users\User\Documents\nav\mmaps_generator.exe" --source-archive "C:\Ascension\Launcher\resources\ascension-live\Data\patch-B.MPQ" --archive "dist\mmaps-335.zip" --descriptor "dist\mmaps-335.json" --archive-url "https://github.com/itszeondk/zeon-assets/releases/download/mmaps-335-v1/mmaps-335.zip"
```

The tool fixes ZIP timestamps, permissions, ordering, and compression settings,
then records the full archive SHA-256 and compressed/unpacked metadata. It
rejects subdirectories, links, unrelated files, invalid 28-byte `.mmap`
records, and `.mmtile` files with the wrong magic or format versions.

### Upload order

Before publishing, confirm that distributing navigation data derived from the
Ascension client is permitted. Do not include `NavSrv.exe`, `Apep.json`, game
data, account information, or the generator itself in the archive.

1. Create the `mmaps-335-v1` release in `itszeondk/zeon-assets`.
2. Upload `dist/mmaps-335.zip` as the release asset named exactly
   `mmaps-335.zip`. Do not put this multi-gigabyte file on the raw branch.
3. Confirm the release-asset URL matches `archiveUrl` in
   `dist/mmaps-335.json` and that its tag is pinned (never `latest`).
4. Copy only the descriptor to `zeon-assets/apep/mmaps-335.json` and push it
   after the release asset is available.
5. Publish the updated `apep/install.ps1` separately through the existing
   installer workflow.

For a changed payload, use a new release tag (for example `mmaps-335-v2`) and
regenerate the descriptor with that pinned URL. Do not replace an archive under
an already-published tag.

The descriptor contains no credentials or machine paths. The installer verifies
its declared archive size and full SHA-256 before staged extraction; it accepts
only flat `.mmap`/`.mmtile` entries, validates header/count/byte invariants,
writes a receipt marker, and then swaps the staged directory into place.
